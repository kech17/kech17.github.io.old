Sensor Analytics

Using sensors, radio modules, and an Arduino to update data to a cloud server in real-time.

The aim of the project was to gather data readings from sensors attached to a remote controlled truck. The sensors will detect behavioural trends in the truck such as accleration, temperature, and rotation. This data is then communicated via wifi module to a cloud server which hosts a dashboard that displays metrics.

skills used.

These are the technical skills learned and used to complete the project.

  • Arduino Microcontroller

    An Arduino Fio V3 microcontroller was used gather sensor data on the truck. C++ was used for code.

  • Sensors and Actuators

    Sensors allow environmental measurements and LEDs server to display behavioural processes of the code.

  • Cloud Computing

    A wifi module posts to a cloud server that analysizes and translates the raw sensor values.

  • Metrics and Analytics

    The cloud server hosts a dashboard page visualizing useful metrics and data trends from sensor values.

Wiring and Soldering

This is the finalized circuit for the project. It consists of 2 LED actuators to show system behaviours. It also has a potentiometer and a digital accelerometer/gyroscope sensor called the MPU6050. The circuit is soldered onto a perf board and custom wiring was used to connect it to the arduino fio v3.

Field Testing with RC Truck

The circuit is fixed on a remote controlled truck for field testing. Sensors pick up accleration, temperature, and rotation readings to determine the state of the truck. For prototype sake, the sensor system is not shielded and fixed on the truck with electric tape.
